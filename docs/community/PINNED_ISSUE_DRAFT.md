## Signal Deck Public Benchmark Tracker (v1)

This tracker defines the benchmark objectives that keep Signal Deck focused on the north star:

**objective -> deterministic plan -> governed execution -> verifiable outcome**

### Why this exists

We want visible, repeatable proof that Signal Deck is improving in reliability, trust, and operator usability.

### Active benchmark objectives

- [ ] Objective 01: Deterministic Planning Path
- [ ] Objective 02: Orchestration Failure Explainability
- [ ] Objective 03: Governance Gate Roundtrip
- [ ] Objective 04: End-to-End DAG Reliability
- [ ] Objective 05: Readiness and Policy Signal Accuracy
- [ ] Objective 06: Data Durability Smoke (Supabase Path)
- [ ] Objective 07: Operator UX Triage Time

### How to contribute

1. Pick an open benchmark issue.
2. Reproduce the baseline result.
3. Propose/implement an improvement.
4. Post before/after evidence in the issue.
5. Keep changes aligned to roadmap phase labels.

### Rules

1. Benchmarks must be reproducible.
2. Each benchmark needs clear pass/fail criteria.
3. Claims without evidence (logs/events/tests) do not count as complete.
4. Prefer objective-to-outcome improvements over unrelated feature additions.
