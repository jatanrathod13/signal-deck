# Security Policy

## Supported versions

Security updates are applied to the `main` branch.

## Reporting a vulnerability

Please do not open public GitHub issues for suspected security vulnerabilities.

Use GitHub Security Advisories for private reporting in this repository:

1. Open the repository on GitHub.
2. Go to `Security` -> `Advisories`.
3. Click `Report a vulnerability`.

## What to include

1. Vulnerability type and impact
2. Affected components and versions
3. Reproduction steps or proof of concept
4. Suggested mitigation (if available)

## Response expectations

1. Initial acknowledgement: within 72 hours when maintainers are available
2. Triage and severity assessment: as soon as practical
3. Fix and coordinated disclosure timeline: based on severity and exploitability
